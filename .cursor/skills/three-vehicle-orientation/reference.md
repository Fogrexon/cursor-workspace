# 回転写像の詳細

## Three.js の Y 回転

右手系。+Y 正方向から原点を見下ろし、反時計が正角。

```text
Ry(θ) = [
  [ cosθ, 0, sinθ ],
  [ 0,    1, 0    ],
  [-sinθ, 0, cosθ ]
]
```

列ベクトルとして `p' = Ry·p`。

### 局所 −Z（機首候補）

```text
Ry(θ)·(0,0,−1) = (−sinθ, 0, −cosθ)
```

### 本プロジェクトの sim forward

```text
forward(θ) := (sinθ, 0, −cosθ)   // pitch=0
```

| θ | forward | Ry(θ)·(−Z) | 一致? |
|---|---------|------------|-------|
| 0 | (0,0,−1) | (0,0,−1) | yes |
| +π/2 | (+1,0,0) | (−1,0,0) | **no（鏡像）** |
| −π/2 | (−1,0,0) | (+1,0,0) | **no（鏡像）** |

### 一致させる ψ

`Ry(ψ)·(0,0,−1) = forward(θ)` より:

```text
(−sinψ, 0, −cosψ) = (sinθ, 0, −cosθ)
⇒ ψ = −θ
```

これが Euler 親ヨーを使う場合の写像。  
**先に − ありきではなく、等式の解。**

## OpenGL / Three.js の「正面」

- カメラは默认で **−Z 方向を見る**
- `Object3D.lookAt` はオブジェクトの **−Z** をターゲットへ向ける
- だから「機体の正面」もローカル **−Z** に揃えると、lookAt / basis と話が通じる

ローカル +Z を機首にすると、親回転や lookAt のたびに ±π の補正が増える。

## basis で組む場合

Three.js `Matrix4.makeBasis(xAxis, yAxis, zAxis)` はローカル軸のワールド向き。

速度 `f` に機首を合わせるなら（ロールなし）:

```text
zAxis = −f    // ローカル −Z = 機首 = f
xAxis = right = normalize(worldUp × f)
yAxis = up    = f × right  （または xAxis × zAxis に注意して一貫させる）
```

ロールは `f` まわりに `xAxis`/`yAxis` を回してから `makeBasis`。

## なぜ「yaw=0 では気づかない」か

写像誤差は `sinθ` の符号差。  
直進（θ≈0）や微小角では機首と速度がほぼ重なり、左右の違和感が出にくい。  
**±90° 旋回で初めて鏡像が明確になる。**  
検証は必ず大ヨー角で行う。

## ロールをセットで反転してはいけない理由

Yaw の写像 `ψ = −θ` は **Y まわりの水平方位**の話。  
ロールは（定義どおりなら）**forward まわり**。  
別グループの回転なので、yaw 写像の符号からロール符号は導けない。  
進行方向を矢印で固定した状態で、翼の上げ下げだけを見る。
